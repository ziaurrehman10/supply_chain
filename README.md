# Supply Chain Risk Co-Pilot

An AI Co-Pilot that scores supply-chain shipments for risk using a
multimodal deep-learning ensemble, explains its predictions, and routes
every recommendation through a human reviewer before it's final.

Built for: **AI Innovation Hackathon — Theme: AI Co-Pilot for Industry
(Track 9: Supply Chain Risk)**

---

## 1. How this maps to the mandatory requirements

| Requirement | Where it lives |
|---|---|
| 3+ data modalities | **Tabular** (`data/supply_chain_data.csv`, the given Kaggle-style dataset) · **Time series** (12-week synthetic lead-time trend per SKU, `synthetic_assets/timeseries.json`) · **Text** (auto-generated supplier/QA notes, `synthetic_assets/supplier_notes.csv`) · **Image** (auto-generated QC risk-gauge PNG per SKU, `synthetic_assets/images/`) |
| Predictive deep learning model | `src/models.py`: **ANN** (tabular risk classifier), **LSTM** (time-series deterioration predictor), **CNN** (image → inspection outcome). Combined in a weighted ensemble. |
| LLM usage restricted to reasoning/summarization | LLMs are **not** used anywhere as the predictive engine. The only optional LLM hook is in `report.py`'s `narrative` argument, for rewording the executive summary — the underlying numbers always come from the ANN/LSTM/CNN ensemble. |
| Human-in-the-Loop | "Human-in-the-Loop Review" page in `app.py` — Approve / Modify / Reject per shipment, persisted to `logs/decisions.csv`. |
| Explainable AI | `src/explain.py` — SHAP feature importance per prediction (falls back to permutation importance if `shap` isn't installed), plus per-model confidence scores shown in the UI. |
| Working web application | `app.py`, a Streamlit app (Dashboard / Risk Predictions / Human-in-the-Loop / Reports / Business Model). |
| Downloadable PDF/DOCX report | `src/report.py` (reportlab) — generated live from the "Reports" page, includes KPIs, risk-distribution chart, and the full HITL decision log. |
| Business model | `BUSINESS_MODEL.md`, also rendered inside the app. |

## 2. Why some modalities are synthetic

The only real dataset provided is tabular (`supply_chain_data.csv`, ~100
SKUs). To honestly satisfy the "3+ modalities" requirement without
fabricating claims about real shipments, the text, time-series, and image
modalities are **generated deterministically from each row's real risk
signals** (defect rate, inspection result, lead time, etc.) — so a "High
risk" SKU gets a red gauge image and a QA-escalation-flavored note, not
random noise. This is disclosed openly here and in the app's "What powers
this dashboard?" expander — be transparent about this in your demo/judging
Q&A. In a production deployment, these would be replaced by: real QC
photos, real carrier/ERP delay logs, and real supplier communication text.

## 3. Setup

```bash
cd supply_chain_copilot
python -m venv venv && source venv/bin/activate   # optional but recommended
pip install -r requirements.txt
```

## 4. Run

**Option A — let the app do everything on first load** (builds synthetic
data + trains all 3 models automatically, ~1-2 minutes the first time,
then cached):

```bash
streamlit run app.py
```

**Option B — build/train explicitly first** (recommended before a demo, so
the live app loads instantly):

```bash
python src/data_pipeline.py   # builds tabular+text+image+time-series data
python src/train.py           # trains ANN, LSTM, CNN and saves them
streamlit run app.py
```

Live link: https://supplychainrisk.streamlit.app/

## 6. Presenting this at the hackathon

Suggested demo flow (fits a 5-10 min slot):
1. **Dashboard** — portfolio-level risk KPIs, 30 seconds.
2. **Risk Predictions** — pick one High-risk SKU, walk through all 4
   modalities feeding into the ensemble score, show the SHAP explanation.
3. **Human-in-the-Loop** — approve one, modify one, reject one; explain why
   this satisfies compliance/audit needs, not just a nice-to-have.
4. **Reports** — generate the PDF live, show the decision log inside it.
5. **Business Model** — 1-slide summary of `BUSINESS_MODEL.md`.

## 7. Known limitations (be upfront with judges)

- Trained on ~100 rows — good enough to demonstrate the full pipeline
  end-to-end, not a production-scale model. Mention the retraining loop
  from `BUSINESS_MODEL.md` (§9-10) as the path to real accuracy.
- Text/image/time-series modalities are synthetic-but-label-consistent
  (see §2 above) since only tabular data was provided.
- SHAP's `KernelExplainer` is model-agnostic but slower than a
  gradient-based explainer; fine for a live demo at this dataset size.
